# MinionMint Product Doctrine Guardrail

MinionMint mints bounded AI workers through a guided blueprint process. Do not implement it as a generic chatbot builder, prompt manager, or widget generator.

Working foundation:

- The first user experience is a minting interview.
- The output is a Minion Blueprint, not a prompt, widget, or generic assistant profile.
- A Minion is a bounded worker with a job, brain, memory rules, operating knowledge, approval rails, first work order, owner review state, and future workstation plan.
- Keep Phase 2 paused until Phase 1 is production-real enough with auth, durable persistence, model-backed blueprint generation, editing, refinement, approval, and operations-board review.
- Use MinionMint-specific language: Minion Blueprint, minting interview, bounded worker, approval rails, first work order, operating knowledge, memory rules, operations board, future workstation.
- Avoid generic chatbot language and broad autonomy. Do not lead with virtual computers, swarms, or computer-use before the Phase 1 blueprint contract is real.
